Clustering with K-Means¶

K-means is one of the simplest clustering algorithms, which you may already know from the course Data Mining. The K-means Algorithm consists in writing a program that proceeds as follows:

  1. Choose value for K (number of clusters);
  2. Initialize the locations of the centroids. This can be done by randomly selecting K points from the dataset as the initial locations of the centroids;
  3. Compute the distance of all the other points to each of the K centroids;
  4. Associate each point to the cluster of the closest centroid;
  5. Update the centroid position. This can be done by computing the average coordinates of all points associated to each cluster;
  6. Evaluate the average change in the centroids positions, as a measure of convergence (the algorithm is said to have converged to the solution when the positions of the centroids do not change more than a given tolerance threshold);
  7. Repeat steps 3-6 until either the centroids no longer move more than a tolerance threshold or until you have repeated these steps for specified number of iterations (niter)

The algorithm is thus parameterized by the number of clusters (K), the maximum number of iterations (niter), and the tolerance threshold (tol).

# The objective is to fetch the centroids for each cluster
def cupdate(points, clusters):
    new_centroids = []
    
    # Fetch the number of clusters (unique numbers)
    num_clusters = len(np.unique(clusters))
    
    # Iterating over each cluster
    for i in range(num_clusters):
        # Filter all the points that belong to the same cluster
        cluster_points = points[clusters == i] 
        
        # Skip empty clusters
        if len(cluster_points) == 0: 
            continue  

        # Calculate the mean of points that belongs to the same cluster, using centroid (step 4) 
        centroid1 = centroid(cluster_points) 
        new_centroids.append(centroid1)

    return np.array(new_centroids)

def pupdate(points, centroids):
    # Create an array to store the cluster index for each point (using np.zeros to fill the array with the correct number of clusters initially)
    new_clusters = np.zeros(len(points))

    # Fetching each point and respective index with enumerate
    for i, point in enumerate(points): 
        distances = []
        for centroid in centroids:
            # For all the points, a distance is calculated (step 3) against each centroid
            distances.append(distance(point, centroid))
            # The index of the lowest distance (the closest centroid) is chosen
            closest_centroid = np.argmin(distances)

        # Assign the closest cluster to the point's index
        new_clusters[i] = closest_centroid

    return np.array(new_clusters)

# The objective is to check the convergy of the method
def error(oldcentroids, newcentroids):
    squared_errors = []
    
    # Calculates distances between old and new centroids
    for i in range(len(oldcentroids)):
        squared_errors.append(distance(oldcentroids[i], newcentroids[i]))
    
    # Computing the mean of all distances
    mean_error = np.mean((squared_errors))
    return mean_error

def kmeans(points, K=2, niter=500, tol=0.00001):
    # Selecting K centroids (from the initial points, randomly)
    centroids = init(points, K)
    
    # Foor loop until reaching the maximum iterations
    for i in range(niter): 
        # Assign each point to a cluster using "pupdate"
        clusters = pupdate(points, centroids)
        # Calculate new centroids using "cupdate"
        new_centroids = cupdate(points, clusters)
        # Calculate error between the old and new centroids using "error"
        change = error(centroids, new_centroids)
        # Check if convergence has been reached
        if change < tol:
            break
        # Update centroids
        centroids = new_centroids
    return clusters, centroids

The Squared of the Euclidean distance is a common measure of error that satisfies several important properties, namely it is strictly convex and smooth. It is often used in the method of Least Squares to fit linear functions to data, in which we try to find the parameters of a linear model that minimize the average of the square distances between the observed values and the values predicted by the fitted model. Here we are borrowing these concepts to measure the dispersion of points in a cluster, in other words, the error between the position of the centroid and the locations of the points associated with it.

# Objective is to calculate inertia of clustering - points dispersion in relation to centroids 
def inertia(points, clusters, centroids):
    # Get number of clusters 
    num_clusters = len(centroids) 
    total_inertia = 0
    
    # For loop to iterate above each cluster
    for i in range(num_clusters):
        # Get all the points assigned to each cluster and respective centroid 
        cluster_points = points[clusters == i]
        centroid = centroids[i]
        # Initialized inside loop because we want to assign the squared distances to each cluster individually (and we are iterating over each cluster)
        squared_distances = []
        # Ignoring clusters with no points
        if len(cluster_points) > 0:
            # For each point inside the respective cluster, calculate the euclidian distance (squared_distance)
            for p in cluster_points:
                 squared_distance = euclidian_distance(p, centroid)
                 squared_distances.append(squared_distance)
            # Inertia for each cluster is the mean of the squared distances calculated
            cluster_inertia = np.mean(squared_distances)
            total_inertia += cluster_inertia

    return num_clusters, total_inertia

As you can see in our function 10, increasing the number of cluster greatly increases the running time of the function as well. So finding the optimal value involves finding a balance between capturing the underlying structure of the data without overcomplicating the model. An optimal choice of K leads to meaningful, interpretable, and stable clusters, enhancing the quality and utility of the clustering results (function itself).
